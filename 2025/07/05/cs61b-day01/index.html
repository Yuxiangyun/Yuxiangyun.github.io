Java的工作流程

在 Java 中，将程序从 .java 文件转换为可执行文件主要有两个步骤：编译和解释。

要在 Hello.java 中运行代码，我们首先需要使用命令 javac HelloWorld.java 将代码编译成一个 .class 文件。

然后，要运行代码，我们需要使用命令 java HelloWorld 。

终端指令
$ javac HelloWorld.java
$ java HelloWorld
Hello World!

Java的静态性

Java和隔壁Cpp一样，都是运行前会检查一下自己有没有错的好孩子，不像隔壁Python只有死到临头才会发现自己错在哪里。

Java 是一种静态类型语言，这意味着所有变量、参数和方法都必须声明类型。声明之后，类型不能更改。表达式也有隐式类型；例如，表达式 3 + 5 的类型为 int。

静态方法 Vs 非静态方法

静态成员（static） 非静态成员（实例成员）
属于 属于整个类 属于某个对象
访问方式 Dog.something dog.something
内存 所有对象共享一份 每个对象都有自己的
创建时机 类加载时就存在 创建对象时才分配

public class Dog {
// ✅ 非静态（实例变量）
String name;
int weight;

// ✅ 静态变量（类变量）
static int numDogs = 0;

// ✅ 构造器
public Dog(String name, int weight) {
    this.name = name;       // 每个对象自己的名字
    this.weight = weight;   // 每个对象自己的体重
    numDogs++;              // 静态变量，记录所有狗的数量
}

// ✅ 非静态方法（实例方法）
public void bark() {
    System.out.println(name + " says: Woof!");
}

// ✅ 静态方法（类方法）
public static int getNumDogs() {
    return numDogs;
}

}

静态方法共享整个类的共有信息，非静态方法用来操作单个对象的状态。

Java 允许我们定义两种类型的方法：
类方法，即静态方法。
实例方法，即非静态方法。

类偶尔需要静态变量。这些变量是类本身固有的属性，而不是实例。
静态变量应该使用类名来访问，而不是特定的实例，例如，你应该使用 Dog.binomen ，而不是 d.binomen 。
虽然 Java 技术上允许使用实例名访问静态变量，但这是一种不良风格，容易引起混淆，并且在我看来是 Java 设计者的一个错误。

揭秘Main方法

public static void main(String[] args)
static：它是一个静态方法，不和任何特定实例关联。
main： 方法的名称。
String[] args：传递给Main的参数。由于 main 是由 Java 解释器本身调用的，而不是由另一个 Java 类调用，因此是解释器的责任提供这些参数。这些参数通常指的是命令行参数。
public class ArgsDemo {
public static void main(String[] args) {
System.out.println(args[0]);
}
}
该程序打印出命令行参数中的第 0 个参数。
